## 🎯 왜 공통을 따로 구분하나요?

### 1. 개발 효율성 ⚙️

**공통으로 만들면:**
```
헤더 컴포넌트 1개만 만들어서
→ 100개 페이지에 재사용

vs.

헤더를 공통으로 안 만들면
→ 100개 페이지마다 헤더 따로 만들어야 함
```

### 2. 일관성 유지 🎨

```
로고를 바꾸고 싶을 때:

✅ 공통으로 만든 경우
   → 헤더 파일 1개만 수정
   → 모든 페이지에 자동 반영

❌ 공통 없이 만든 경우
   → 100개 페이지 파일 전부 수정
   → 실수로 빠뜨리면 페이지마다 로고가 다름
```

### 3. 사용자 경험 개선 👤

```
모든 페이지에서 같은 위치에:
- 로고 → 클릭하면 홈으로
- 검색 → 어디서든 모임 검색 가능
- 내 정보 → 어디서든 프로필 확인

→ 사용자가 헤맬 필요 없음!
```

## 📋 WOORIDO 공통 요소 상세

### 1. 헤더 (Header) - 모든 페이지 맨 위

```
┌──────────────────────────────────────────┐
│ 🏠 [로고]                                │
│    클릭 → 홈으로 이동                     │
├──────────────────────────────────────────┤
│ 🔍 [검색창]                              │
│    키워드 입력하면 모임 검색              │
├──────────────────────────────────────────┤
│ 📱 [메뉴]                                │
│    - 홈                                  │
│    - 모임 찾기                            │
│    - 나의 모임                            │
├──────────────────────────────────────────┤
│ 👤 [사용자 메뉴]                          │
│    • 로그인 전: [로그인] [회원가입]       │
│    • 로그인 후:                          │
│      - 닉네임: 홍길동                     │
│      - 잔액: 100,000원                   │
│      - [충전하기] 버튼                    │
└──────────────────────────────────────────┘
```

#### 반응형 (모바일 vs 데스크탑)

**데스크탑 (≥768px):**
```

│ 🏠 WOORIDO    홈  모임찾기  나의모임    🔍 검색 │
│                              홍길동님  10만원  │
│                                     [충전하기] │
└────────────────────────────────────────────────┘
```

**모바일 (<768px):**
```
┌──────────────────┐
│ ☰  🏠 WOORIDO  🔍│  ← 햄버거 메뉴로 축소
└──────────────────┘

클릭 시:
┌──────────────────┐
│ ☰ [메뉴]         │
│ ───────────────  │
│ • 홈             │
│ • 모임 찾기       │
│ • 나의 모임       │
│ • 내 정보         │
│ • 충전하기        │
└──────────────────┘
```

### 2. 레이아웃 (Layout) - 페이지 구조

```
공통 레이아웃이 제공하는 것:

✅ 반응형 처리
   - 모바일: 세로 스택 (한 줄씩)
   - 데스크탑: 가로 배치 (여러 열)

✅ 글로벌 오류 처리
   - 네트워크 끊김 → "인터넷 연결을 확인해주세요"
   - 세션 만료 → 자동으로 로그인 페이지 이동
   - 서버 오류 → "잠시 후 다시 시도해주세요"
```

**예시:**
```
모든 페이지에서 와이파이가 끊기면
→ 레이아웃이 자동으로 감지
→ 팝업 띄움: "⚠️ 인터넷 연결을 확인해주세요"
```

## 🤔 자주 묻는 질문

### Q1. 모든 페이지에 헤더가 있나요?
**A.** 네! 로그인 페이지, 홈, 모임 상세, 대시보드... 모든 페이지 맨 위에 똑같은 헤더가 있습니다.

---

### Q2. 헤더를 숨길 수 있나요?
**A.** 특수한 경우에만 가능합니다:
- 회원가입 Step 페이지 (집중 필요)
- 결제 페이지 (보안)
- 풀스크린 모드

일반 페이지에서는 항상 표시됩니다.

---

### Q3. 공통 vs 페이지 구분이 헷갈려요
**A.** 간단한 기준:

```
✅ 공통:
   "모든 페이지에서 똑같이 보이는 부분"
   예: 로고, 메뉴, 잔액 표시

❌ 공통 아님:
   "페이지마다 달라지는 부분"
   예: 모임 목록, 검색 결과, 대시보드 내용
```

---

### Q4. 공통을 먼저 만들어야 하나요?
**A.** 네! 개발 순서:

```
Week 0-1: 공통 (헤더/레이아웃/푸터) 먼저 만들기
    ↓
Week 2-3: 개별 페이지 만들기 (로그인, 홈, 모임 생성...)
    ↓
각 페이지는 공통 레이아웃 안에 들어감
```

## 🎨 실제 코드 구조 (참고용)

```typescript
// 공통 레이아웃 컴포넌트
function Layout({ children }) {
  return (
    <div>
      <Header />        {/* ← 공통: 모든 페이지에 */}

      <main>
        {children}      {/* ← 페이지마다 다른 내용 */}
      </main>

      <Footer />        {/* ← 공통: 모든 페이지에 */}
    </div>
  )
}

// 각 페이지에서 사용
<Layout>
  <HomePage />        {/* 홈 내용만 다름 */}
</Layout>

<Layout>
  <SearchPage />      {/* 검색 내용만 다름 */}
</Layout>

<Layout>
  <DashboardPage />   {/* 대시보드 내용만 다름 */}
</Layout>
```

## ✅ 요약

> **공통 (Global) = 모든 페이지의 "뼈대"**

```
┌─────────────────────┐
│   헤더 (공통)        │  ← 모든 페이지 공통
├─────────────────────┤
│                     │
│   페이지 내용        │  ← 페이지마다 다름
│   (페이지마다 다름)   │
│                     │
├─────────────────────┤
│   푸터 (공통)        │  ← 모든 페이지 공통
└─────────────────────┘

공통을 먼저 만들면:
→ 100개 페이지를 만들어도 헤더/푸터는 1번만 작업
→ 일관성 유지
→ 개발 효율 극대화
```
